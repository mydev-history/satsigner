# Satsigner Versioning

## App Version & Build Number

Satsigner uses [react-native-device-info](https://github.com/react-native-device-info/react-native-device-info) library to get information about the app version and build number.

The app version follows semantic versioning.

In android, it can be found in `android/app/build.gradle` file on `android.defaultConfig.versionName`:

```gradle
android {
  defaultConfig {
    versionName "0.1.0"
  }
}
```

In iOS, in `ios/satsigner/Info.plist`:

```
<key>CFBundleShortVersionString</key>
<string>0.1.0</string>
```

The build number can be found in the same files. For android, search for `versionCode` and for iOS search for `CFBundleVersion` key.

You can define these properties in `app.json` file:

```json
{
  "expo": {
    "version": "0.1.0",
    "android": {
      "versionCode": "1"
    },
    "ios": {
      "buildNumber": "1"
    }
  }
}
```

Read more about App versions [here](https://docs.expo.dev/build-reference/app-versions/#app-versions)

## Managing Manually

To update `app version` and `build number` manually, just modify the numbers in `app.json` file and run prebuild for both `android` and `ios`.
